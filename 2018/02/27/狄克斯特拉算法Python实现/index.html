狄克斯特拉算法Python实现

Feb 27, 2018 | python编程

一，概念介绍

  狄克斯特拉算法（Dijkstra）可以用于查找加权有向图中从起点到目的地的最短距离，与广度搜索优先不同的是，该算法是通过计算从起点到目的地的各个路径的权重，来选出开销最小的路径，而广度搜索则是通过计算两点之间的节点数来判断是否是最短路径。

二，代码实现

  使用字典（散列表）来表示图的节点和边，通过字典中Value值为子字典的方式来表示邻居节点，以及从当前节点到邻居的权重值。采用空字典做图的终点。
实现思路
 1.使用graph字典将整个图的node边以及weight值表示出来，即是构建一个图
 2.使用costs字典记录起始点到所有点的开销（图计算完成后，里面都是最小开销），初始值为起点到它的邻居节点的值。
 3.创建一个parents字典记录该节点的父节点。可用于绘制关系图
 4.创建一个函数用于找出costs中没被用过且开销最小的值，第一次执行为起点最近的邻居
 5.定义一个用于计算的函数如下：
  (1)第一次在costs中找出起点邻居用作初始值
  (2)然后使用一个循环，将costs中的每个node都进行如下操作：
    1.找到当前node的权重
    2.找出当前节点的邻居
    3.起始点到当前节点的权重+当前节点到邻居节点的权重=起始点到该邻居节点的权重
    4.判断并更新或者增加邻居节点的总开销，并将邻居节点加入costs, 然后将当前节点加入到已经操作的字典。无邻居节点就不进行操作
    5.对剩余的每个节点进行同样的操作（检查更新邻居节点）,直到graph中的节点被计算完才结束

三，总结

1.狄克斯特拉算法不能用于有负权重的图当中
2.用于有环的图中（俩节点间双向箭头）没有任何意义，绕环会增加权重
3.常用于查找俩点间带权重考量的最短路径

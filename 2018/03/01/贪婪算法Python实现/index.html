Mar 1, 2018 | python编程

一，概念介绍

贪婪算法每次从可选的未被选中过的选项中选择最大的或者最优的，依次重复该操作，直到达到上限，得到的是一种近似解。适用于：精确计算时计算量比较大，但是要求速度快且计算结果可以是近似最优解的情况。运行时间为O(n^2)。每次都是求出局部最优解，然后组合起来。
贪婪算法是一种近似算法，衡量近似算法优劣的标准：
1.速度
2.与最优解的接近程度

三，总结

1.贪婪算法可用于集合覆盖或旅行商问题这类NP完全问题的求近似解
2.NP问题计算量太大，时间动则很多年或者无法完全计算出来，只能求近似解
3.狄克斯特拉与广度搜索优先解决的是点到点的最短路径问题，贪婪算法解决的是复杂的覆盖问题或者序列问题。得到近似解

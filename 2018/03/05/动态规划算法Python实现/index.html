笔记

1.概念

1.算组合的算法运行时间为O(2^n)，如果元素比较多，就很大了。动态规划从小问题入手，逐步解决大问题，合并多个子问题的解来得到更大问题的解
2.每次迭代时，你都存储当前的最大价值。行的排列顺序不重要
3.动态规划必须处理的是整件的问题，不能处理取部分货物的问题，可用贪婪算法解决这类问题
4.仅当每个子问题都是离散的，即不依赖于其他子问题时，动态规划才管用
5.最多只需合并两个子背包，子背包下可能又包含子背包

2.优势

1.动态规划可帮助你在给定约束条件下找到最优解。每种动态规划解决方案都涉及网格
2.在问题可分解为彼此独立且离散的子问题时，就可使用动态规划来解决。
3.单元格中的值通常就是你要优化的值。在前面的背包问题中，单元格的值为商品的价值。
4.每个单元格都是一个子问题，因此你应考虑如何将问题分成子问题

3.解决思路

解决动态规划问题的思路：
 单元格中的值是什么？
 如何将这个问题划分为子问题？
 网格的坐标轴是什么？

4.费曼算法：

(1) 将问题写下来。
(2) 好好思考。
(3) 将答案写下来。

5.应用场景

在动态规划中，你要将某个指标最大化，要找出的是最长公共子串，可以解决用户输入单词矫正等问题。
最长公共子序列

最长公共序列可解决的问题：
1.git diff的判断文件差异的问题
2.生物学用于DNA链的相似性
3.字符串相似度比较

ELK笔记之Elasticsearch关键概念

Feb 25, 2019 | 运维杂记

一、分片相关

1.一个分片是一个Lucence实例，分片本身就是一个完整的搜索引擎。
2.分片是数据的容器，文档被存储及索引到分片内。
3.分片分为主分片和多个副本分片,副本分片用于冗余及搜索和返回文档等读操作，索引建立时确定主分片数，副本分片可以随时修改，默认情况索引会被分配5个主分片。
4.一个主分片可以存储的文档数:Integer.MAX_VALUE-128。

二、倒排索引

1.在lucence中，一个索引是放在一个文件夹的
2.segment就是倒排索引，是不可变的，段是用来管理文档的，比如一个索引有多少个段，shard只是为了便于存
储负载。
3.倒排索引用于快速全文搜索，一个倒排索引由文档中所有不重复的词的列表构成，有一个文档列表，索引检索算法是用空间来换取时间。
4.一个doc就是一条日志记录，等同于关系型数据库的一条数据记录，elsticsearch中文档是有字段和值的JSON文档，每个被索引的字段都有自己的倒排索引。
5.filed就是域，可以分开索引，比如标题，时间等，都可以保存在不同的域里面，指的是文档中的各个字段。
6.Term词是索引的最小单位，是经过词法分析后和语言处理后的最小字符串，某个词，搜索时的一个单位，文本中的某个词。

三、其余

1.建索引:把文档信息写到buffer中(也就是segment缓存文件)，然后flush刷入磁盘。
2.更新时，老数据会打上已删除标记，在fulsh的时候，才会物理删除
3.yelllow状态是主分片正常，副本没有完全处于正常状态，单机的话是yellow，主副在一台上没意义
4.token就是指的词条信息

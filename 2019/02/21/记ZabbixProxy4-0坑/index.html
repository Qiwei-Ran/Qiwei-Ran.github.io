记升级到ZabbixProxy4.0坑

Feb 21, 2019 | 运维杂记

   一、问题及解决

   1.zabbix_proxy升级按照官方文档步骤操作即可，此处省略，但是建议做好配置文件的备份，以防不可预测的问题。
   2.在zabbix_proxy服务器上的zabbix_agent会自动随着Proxy一起升级到4.0(yum方式)，配置文件都会自动替换，需要注意的是，老版本ZabbixProxy配置文件中的HistoryTextCacheSize已经被废弃，启动会报错， 同位置被HistoryIndexCacheSize选项替换，作用是用于设定历史书库索引缓存的大小。
   3.重点需要注意的是，zabbix_proxy2.2升级后，会发现有部分的主机都会报agent.ping 不可达时间达到5分钟，server端和proxy端日志无任何异常,查看agent日志，报错如下:

得知是跟TCP连接相关，后来经过大量的尝试以及代码查看，最终问题确定，是由于在升级后，proxy服务器上必须要将被监控的服务器agent配置文件中的Hostname与IP写入到hosts文件中，不然zabbix连接会建立失败。
   4.在处理上面个问题是发现一个新问题，就是被监控的主机如果时间和zabbix server的时间对不上的话，告警会误报，体现就是数据图形都是连续的，但是告警频繁波动，时好时坏，这种情况调整到正确的时间就可以解决。
   4.Agent端有多IP的时候注意下，若在Proxy加了hosts后还是出现如上报错，将proxy中hosts中IP换成另外一个即可解决。

   二、收获

1.自己在排查这一块的问题，只是简单的通过telnet检查连通性就排除了网络的问题，这说明在处理问题这一块儿还需要更多的经验积累。
2.要对TCP协议更加深入的研究，以及对程序报错的相关代码进行查阅才能更好的运维。
3.在思维陷入一个深坑的时候，不妨换种方式，或许就可以找到解决方案。
